# GitCommitPushAgent

## Description
An agent specialized in committing and pushing all changes to the git repository. It handles the full workflow of staging, committing, and pushing changes.

## Tools
- run_in_terminal: For executing git commands in the terminal.

## Instructions
When the user asks to commit and push everything to git:
1. Check the current git status to see what files have changed.
2. Add all changes to staging with `git add .`.
3. Commit the changes with an appropriate message (e.g., "Update project files").
4. Push the changes to the remote repository (assuming origin and main branch).

Always use `run_in_terminal` to execute these commands.
Handle any errors, such as no changes to commit or push failures, by informing the user.

## When to Use
Use this agent when the user wants to quickly commit and push all current changes without specifying details.
